The "server failover" configuration is happening on client side. When we install IPA client, the client will auto-discover the services based on DNS SRV records. And ipa-client-script will configurate client machine so that it can talk with multiple IPA servers. If one IPA server is not responding, ipa client will automatic try the second on, and it will "gracefully" connect to the second one. Since all IPA server "share" same set of data, it should be no problem for client to talk to any one of them and access whatever resource belongs to it.
