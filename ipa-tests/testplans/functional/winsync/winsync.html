PRD request

IPA 1.0 PRD has the following requests:

  • [Req6.1] The IPA server should support all the Replication features of RHDS8.0. Including 4 way multimaster replication and Windows Sync.
  • [Req6.5] Documentation should be provided to simplify Windows Active Directory integration and synchronization
  • [Req8.1] Red Hat Directory Server 8.0 on RHEL 5.1
    • The WindowsSync feature of DS will be tested/supported on AD Windows Server 2003, AD Windows Server 2000
  • [Req11] Windows interop
    • Windows Platform support falls into 1 category
    • [Req11.1] AD clients: The windows client will rely on Microsoft Active Directory for account information and Authentication services. The IPA server will use the Windows Sync functionality to synchronise Username, Password and Group information. This scenario covers 3 separate use cases that will be supported
      • [Req11.1.1] Win sync between AD and IPA only
      • [Req11.1.2] Kerberbos trust relationship between AD and IPA only --How this one being implemented?
      • [Req11.1.3] Both Win sync and Kerberbos trust relationship between AD and IPA --How does this feature being implemented?
  • [Req23.6.2] Windows user account management tools on supported Windows platforms in the following scenarios --what is Windows user account management tools?
    • If windows sync is installed on an Active Directory domain controller.

Test Cases

Test cases fall into 2 big categories:

  1. create a user on ipa server (or replicant),
    1. this user entry should sync to window 2003 server
    2. this user should do system login to window 2003 server
  2. create a user on window 2003 server
    1. the user entry should sync to ipa server and its replicants.
    2. In addition to entry synchronization, this user should also be able to access all system resource as others that created originally on ipa server.
      1. system login
      2. obtain kerberos credential
      3. ssh access
      4. NFS mounting
    3. Password policy enforcement should apply to window server as well.


TC# scenario execution
1 create user on "ipaserver", user entry should appears on window 2003 server
2 create user on "ipaserver", user should be able to system login on window 2003 server
3 create user on "win2003" server, user should be able to do system login on ipa server (both rhel5 & rhel 4, 32 & 64 bit)
4 create user on "win2003" server, user should be able to obtain kerberos ticket from ipa server/replica in any ipa client host
5 based on #4, user should be able to login to ipa server without password prompt via ssh
6 based on #4, user should be able to mount NFS that available to this user
7 modify user password on win2003 server: if new password violates the existing password policy on ipa server, new password won't be accepted
8 modify user password on ipa server: if new password violates the existing password policy on win2003 server, new password won't be accepted Am I right here?
9 'memberOf' testing in the context of Windows Sync. Do add more testcases for create/modify/delete groups and its members. Do the same from the AD side. Final verification for group membership can be done via ipa cli's or getent type commands. Test case details on replica test plan
